# Dataset Organization Structure

## Raw Data Directory (`data/raw/`)

### Dataset-Specific Folders:

```
data/raw/
├── faceforensics/          # FaceForensics++ (requires form approval)
│   ├── README.md           # Dataset documentation and citations
│   ├── src/               # Source utilities and scripts
│   └── assets/             # Demo images and videos
├── celeb-df/               # Celeb-DF (requires form approval)
│   └── [place dataset here when approved]
├── dfdc/                   # Facebook Deepfake Detection Challenge
│   └── [place dataset here when downloaded from Kaggle]
└── synthetic/               # Generated synthetic data for testing
    └── [auto-generated by preprocessing scripts]
```

## Processed Data Directory (`data/processed/`)

### Training-Ready Structure:

```
data/processed/
├── train/
│   ├── real/              # Real/authentic media
│   └── fake/              # Fake/synthetic media
├── val/
│   ├── real/              # Validation real samples
│   └── fake/              # Validation fake samples
└── test/
    ├── real/              # Test real samples
    └── fake/              # Test fake samples
```

## Data Flow

### Pipeline Process:
1. **Download** → `data/raw/[dataset]/`
2. **Preprocess** → Face detection, normalization, resizing
3. **Split** → Train/Val/Test (70/15/15%)
4. **Train** → PyTorch models with validation
5. **Export** → ONNX format for browser deployment

### Browser Deployment:
- **Models**: Exported to `models/` as ONNX files
- **Size**: Lightweight models < 0.1MB for web deployment
- **Format**: ONNX Runtime Web compatible
- **Inference**: <100ms per image on CPU

## Getting Started

### Immediate (Available Now):
```bash
# Use synthetic data for testing
python src/train_pytorch.py test

# Use DFDC dataset (requires Kaggle download)
# 1. Download from: https://www.kaggle.com/c/deepfake-detection-challenge
# 2. Extract to: data/raw/dfdc/
# 3. Run preprocessing
python src/preprocessing.py --data_dir data/raw/dfdc
```

## Recommended Dataset Priority

### For Development:
1. **DFDC** (Kaggle) - Large, diverse, immediately available
2. **Synthetic** - Quick testing, pipeline validation

This structure ensures:
